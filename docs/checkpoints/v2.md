# V2 rebuild — checkpoint (P0–P5)

**The reset:** iteration one (M0–M8, engine model) shipped as v0.1.0-beta and was composted on the founder's call — five engines were four too many. Every medium shares one story skeleton; mediums are presets (data), not engines (code). The design, the plain-files promise, and ~70% of the code survived intact; the engine system did not.

## What was built

1. **Preset system** — `PresetDefinition` (terminology, structure, defaults, scaffold, medium tag, home, suggested plugins); ~20 built-in presets as core data; `project.json` stores `{ preset, medium }`; scaffold-driven project creation; new-project gallery grouped by medium.
2. **Plugin API v2** — no engines; `presets.register` (zero-code community media); `ExporterDefinition.media` (the join for the export dialog); editor extension factories gated per-editor by `{ filePath, medium, preset }`; `@codemirror/*` allow-listed in the require shim (app's own instances).
3. **Universal exports** — Word, EPUB, PDF, clean Markdown, single-file HTML, Markdown zip, built in for every project via `packages/exporters`; media-join dialog merges medium plugins' formats.
4. **Medium plugins** (`plugins/seed/`): screenplay-mode (element inference + Tab cycling + Courier mode), comic-rail (PAGE/PANEL decorations + vanilla flow rail + insert commands), story-canvas (vanilla SVG canvas + walk; project-relative `node.file` fixes a v1 export-path bug), medium-exporters (Fountain/FDX/screenplay PDF, comic script, Ren'Py/Twee/outline/JSON, GM-visibility honored).
5. **Companion seeds** (lablooms.* ids): structure-wizards, sprints, namesmith, stats-deluxe. Extra Exporters retired (universal covers it).
6. **Docs** — SPEC v2, AGENTS v2, fresh CHANGELOG, README, user guide, plugin dev docs with typedoc API reference, decisions.

## Verified

- Gates: `pnpm lint`, `typecheck`, `test` (126 unit), `test:registry`, `build` — green.
- e2e: 29 specs green locally — including community install → trust → Courier screenplay mode → Fountain export, and theme install → live repaint (release assets route-stubbed until the push publishes them).
- Packaged Windows build verified (see below).

## Honest gaps

- Medium plugins were ported by subagents and smoke-verified; their interactive feel (canvas dragging, rail rhythm, screenplay cycling) deserves one human pass — the e2e covers activation, not delight.
- Registry repo still holds v1 entries until the push updates it (v2 fixture is in-repo; the app falls back to it offline).
- Release assets for v2 seeds/themes don't exist yet — published at push time alongside the v0.1.0-beta release.
- Old seed/theme releases and the engine-model v0.1.0-beta were deleted from GitHub on the founder's word; iteration one survives only as a local-only archive branch.

1. **Writing feel (W4)** — Tab/Shift-Tab indent & outdent (never focus-escape), Enter continues lists, brackets pair while quotes curl, scene breaks centered. `e2e/writing-feel.spec.ts` (4).
2. **Codex (W1)** — the real bug found: CM's FuzzyMatcher only matches 1-char queries at a label's first character, so `@x` killed the tooltip (labels started with `@`). Fixed at the label; panel gained filter, current-project creation default, live refresh, no-container guidance. `codex.spec.ts` +2.
3. **Offline installs (W2)** — `bundled` registry entries install first-party seeds/themes from the app bundle (allow-listed `app.readBundled`, extraResources); remote merges bundled-first; third-party download path kept and e2e-proven (stubbed Echo Garden).
4. **App-iness (W3)** — Help menu (About / Check for Updates… / User guide / Report an issue / DevTools), About dialog with the single pink credit, manual update check with three designed outcomes, full preset icon map, `window.toggleDevTools` bridge.
